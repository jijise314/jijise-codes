# 피보나치 수열과 GCD 알고리즘 시간 복잡도 분석

### 해당 코드는 long lomg 자료형의 특성상 92가 최댓값입니다.

## 1. 알고리즘 구성 요소
1. $n=5$부터 $N$까지 반복하는 외부 루프 (for문)
2. 루프 내부에서 매번 호출되는 유클리드 호제법 (get_gcd 함수)

시간 복잡도를 계산하기 위해 기준이 되는 입력값을 $N$ (피보나치 수열 최대 항의 개수)으로 정의합니다.

### 유클리드 호제법의 시간 복잡도
`get_gcd(a, b)` 함수는 `a % b` 연산을 반복하여 나머지가 0이 될 때까지 실행됩니다. 일반적으로 유클리드 호제법의 시간 복잡도는 $O(log(min(a, b)))$ 입니다.

하지만 **연속된 두 피보나치 수열**을 유클리드 호제법에 넣을 때는 아주 특별한 수학적 현상(라메의 정리, Lamé's Theorem)이 발생합니다.

$F(n) = F(n-1) + F(n-2)$ 이므로, $F(n)$을 $F(n-1)$로 나눈 나머지는 항상 정확히 $F(n-2)$가 됩니다.

따라서, $F(n)$과 $F(n-1)$의 GCD를 구할 때 `get_gcd` 함수의 시간 복잡도는 $O(n)$이 됩니다.

### 외부 루프의 시간 복잡도
외부 루프는 $n = 5$부터 $N$까지 총 $N-4$번 반복 실행됩니다. 루프 자체만 놓고 보면 $O(N)$의 시간이 걸립니다.


## 2. 전체 시간 복잡도 (Big-O) 계산

전체 프로그램의 실행 시간은 외부 루프가 도는 횟수와 내부 GCD 함수의 실행 시간의 합으로 계산할 수 있습니다.

$n$이 5부터 $N$까지 커질 때, 각 루프에서 수행되는 나눗셈 연산의 횟수 합을 수식으로 표현하면 다음과 같습니다.

$$T(N) = \sum_{n=5}^{N} O(n) = O(5) + O(6) + O(7) + \dots + O(N)$$

이는 등차수열의 합 공식에 의해 근사치로 $\frac{N(N+1)}{2}$에 비례하게 됩니다. Big-O 표기법에서는 상수항과 최고차항 미만의 항을 무시하므로, 최종 시간 복잡도는 $N^2$이 됩니다.  

최종 시간 복잡도: $O(N^2)$
