# 피보나치 수열과 GCD 알고리즘 시간 복잡도 분석

## 피보나치 수열의 시간복잡도
피보나치 수열을 수식 $F(n) = F(n-1) + F(n-2)$ 그대로 구현할 경우, 한 번 호출 시 2개의 함수가 추가로 호출되는 트리 구조를 형성합니다.  
이 과정에서 동일한 값을 수없이 중복 계산하게 되어 시간 복잡도가 지수 시간인 $O(2^n)$으로 기하급수적으로 폭발합니다.  

따라서 해당 알고리즘 문맥에서 피보나치 수열의 시간복잡도는 $O(2^n)$ 입니다.

## 최대공약수(GCD)의 시간복잡도
최대공약수를 구하는 알고리즘에 적용된 방법은 유클리드 호제법입니다. 일반적인 유클리드 호제법의 시간 복잡도는 $O(\log(\min(a, b)))$ 입니다.  
### 현재 코드 문맥에서의 시간 복잡도는 라메의 정리를 적용받아 $O(n)$ 입니다.
- main 함수에서는 get_gcd의 인자로 인접한 두 피보나치 수 $F_n$과 $F_{n-1}$을 전달하고 있습니다.
- 유클리드 호제법에서 가장 많은 나눗셈을 수행하는 최악의 조건이 바로 두 인자가 인접한 피보나치 수일 때입니다(라메의 정리). 이 경우 나눗셈 연산은 정확히 $n-2$번 수행됩니다.  

따라서 현재 알고리즘 문맥에서 get_gcd의 시간 복잡도는 $O(n)$이 됩니다.
