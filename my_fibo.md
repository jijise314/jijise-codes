# 피보나치 수열 알고리즘 성능 프로파일링 및 비교 분석

## 1. 알고리즘 구현 코드

### 1-1 재귀적 방법(recursive)
함수 자신을 다시 호출하는 재귀를 사용하여 피보나치 수열의 수학적 정의를 직관적으로 구현한 방식입니다.

unsigned long long fib_recursive(int n) {
    if (n == 1 || n == 2) {
        return 1;
    } else {
        return fib_recursive(n - 1) + fib_recursive(n - 2);
    }
}

### 1-2 순환적 방법(loop)
반복문을 사용하여 이전 두 항의 합을 다음 항으로 갱신해 나가며 값을 구하는 방식입니다.

unsigned long long fib_loop(int n) {
    if (n == 1 || n == 2) {
        return 1;
    } else {
        unsigned long long fn1 = 1, fn2 = 1, temp = 0;
        for (int i = 3; i <= n; i++) {
            temp = fn1 + fn2;
            fn1 = fn2;
            fn2 = temp;
        }
        return temp;
    }
}

## 3. 알고리즘 분석 및 비교

### 3.1 시간 복잡도 비교
- **재귀적 방법: *O*(2^N)** <br>
함수가 한 번 호출될 때마다 내부에서 두 개의 함수를 추가로 호출합니다. 이미 계산한 하위 피보나치 항을 기억하지 못하고 **계속해서 중복 계산**을 수행하므로, N이 커질수록 연산량이 지수 형태로 폭발적으로 증가합니다.
- **순환적 방법: *O*(N)** <br>
반복문을 N-2번 순회하면서 값을 단 한 번씩만 계산합니다. 중복 연산이 전혀 발생하지 않으므로 데이터가 커져도 처리 시간이 선형적으로 일정하게 증가하며 매우 빠릅니다.

### 3.3 최종 결론
재귀적 방법은 순환적 방법에 비해 시간적으로 상당한 성능 저하가 발생합니다. 따라서 피보나치 수열은 재귀적 방법보다 순환적 방법을 사용하는것이 효율이 압도적으로 좋습니다.
